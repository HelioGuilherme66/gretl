[gretl main page]

Gretl: Gnu Regression, Econometrics and Time-series Library

for Mac OS X

Step 0

At some points below, the appropriate choice depends on what sort of Mac you're running. If you're unsure about this, click on the Apple icon at the top left of your screen and select "About this Mac". In the window that appears you'll find the OS X version number and also the processor version, Intel or PowerPC (PPC).

Step 1

To run gretl, you need to have the X11 window system installed on your Mac. This should be automatic on recent OS X, but if you're using an earlier version you may have to go back to the installation DVD to install X11.

How to tell if X11 is installed? Open the Finder and go to Applications. Under Applications, find the Utilities folder, open it and scan to the end: if X11 is installed it should appear there as an icon.

Step 2

Some additional libraries are needed to support gretl, namely GTK and friends. So, unless you have already done so, install the GTK+ Framework for OS X (or alternate download).

Step 3

Now for gretl itself: grab and install either gretl-VERSION-intel.dmg.gz for an Intel-based Mac, or gretl-VERSION-ppc.dmg.gz for a PPC Mac.

But wait... the gretl links above are to the most recent official release. Another possibilility is to download the current snapshot of gretl: either gretl-intel.dmg.gz for Intel or gretl-ppc.dmg.gz for PPC. This will be more up to date; sometimes it will contain bug-fixes, but sometimes it will contain newly introduced bugs. Translations into languages other than English are likely to be incomplete. What's new in the snapshot? Take a look at the Change log (the "in progress" entry).

The gretl disk image (dmg) is briefly documented in README.pdf, which contains some suggestions for trouble-shooting. Briefly, if you want to debug a non-functioning gretl on OS X, please try this: open an X11 terminal window; change directory (cd) to Gretl.app (where you installed the package); change directory to Contents/Resources/bin and run the command

./gretl

This should give you some feedback on what is going wrong.

Optional extras
