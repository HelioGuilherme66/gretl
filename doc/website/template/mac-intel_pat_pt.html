Gretl: Gnu Regression, Econometrics and Time-series Library

para Mac OS X com processador Intel (versão X11)

Passo 0: se necessário instalar X11, (apenas na primeira vez)

Se você usa OS X 10.8 (Mountain Lion), e ainda não tiver instalado o X11, terá que fazer isto primeiro. (X11 vinha instalado por omissão no OS X 10.6 e 10.7.) Você tem que obter o X11 a partir de macosforge.

Passo 1: instalar GTK (apenas na primeira vez)

Algumas bibliotecas adicionais são necessárias para o gretl, nomeadamente GTK e outros relacionados. Se você estiver usando OS X 10.6 (Snow Leopard) ou superior, instale GTK usando GTK2-framework.pkg. Algumas notas sobre este pacote podem ser encontradas em GTK.pdf.

Se você estiver usando uma versão mais antigo do OS X (10.5 ou anterior) você deve instalar esta versão do GTK (ou sítio alternativo). Atenção: Esta versão do GTK não funcionará no OS X 10.7 (Lion).

Passo 2: instalar gretl

Agora para o próprio gretl: obter e instalar a versão mais recente ou o "snapshot" (versão de desenvolvimento). O "snapshot" está mais atualizado: normalmente conterá correções de erros, mas por vezes poderá ter novos erros introduzidos. Para saber o que existe novo no "snapshot", dê uma vista de olhos no Registo/Registro de alterações do gretl (na entrada "in progress").

versão mais recente (SHORTDATE)

gretl-VERSION-x11.dmg

OU "snapshot" atual

gretl-x11.dmg

Depois de obter o gretl, faça um duplo-clique no arquivodmg para montar a imagem de disco. Abrir a imagem de disco no Localizador e arrastar Gretl para dentro de Aplicativos. Abrir a imagem de disco no Finder e arrastar Gretl para dentro de Applications. Mas note: se você estiver atualizando uma instalação do gretl, você deverá primeiro abrir Aplicativos com o Localizador e apagar o item do Gretl antigo.

Depois de ter instalado o gretl você pode ejetar a imagem de disco e apagar o arquivo dmg.

Problemas?

A imagem de disco do gretl disk image está documentada em README.pdf, que contém algumas sugestões para despiste de problemas. Resumidamente, se você quiser despistar problemas num gretl não funcional em OS X, tente isto: abrir uma janela de terminal X11; mudar de diretório (cd) para Gretl.app (o local onde instalou o pacote); mudar para o diretórioContents/Resources/bin e rodar o comando

./gretl

Isto deve mostrar alguma informação sobre o que está acontecendo de errado.

Extras opcionais

Você pode querer instalar X-12-ARIMA e/ou TRAMO/SEATS. Estes são programas especializados para análise de séries temporais com sazonalidade, em versões compatíveis com gretl.
