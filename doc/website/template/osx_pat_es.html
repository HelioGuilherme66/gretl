para Mac OS X

Imagen de disco pre-compilada

La forma más simple de instalar gretl para OS X es grabar e instalar dos ficheros de imagen (dmg) siguiendo los pasos:

  • En primer lugar, instalar el paquete GTK+ Framework para OS X (o desde este otro sitio)

  • Luego, grabar e instalar uno de los siguientes gretl-VERSION-intel.dmg para un Mac basado en Intel, o gretl-VERSION-ppc.dmg para un Mac PPC.

Los enlaces de arriba a gretl apuntan a la última versión oficial publicada. Otra posibilidad es descargar la versión "instantánea" de gretl: o gretl-intel.dmg.gz para Intel o gretl-ppc.dmg.gz para PPC. Es probable que ésta esté más actualizada: a veces contendrá correcciones de errores, y otras veces ¡contendrá errores nuevos! Las traducciones a idiomas diferentes del Inglés es probable que estén incompletas. ¿Qué hay de nuevo en la versión "instantánea"? para saberlo, eche un vitazo al historial de cambios (la lisa de cosas "en preparación").

La imagen de disco de gretl está documentada abreviadamente en README.pdf, que contiene algunas sugerencias para resolución de problemas. En breve, si Vd desea chequear los errores de un gretl sobre OS X que no funciona, por favor intente esto: abra una ventana de terminal X11; cambie el directorio (cd) a Gretl.app (donde Vd. instaló el programa); cambie el directorio a Contents/Resources/bin y ejecute la instrucción

./gretl

Esto debería de darle algo de información sobre qué es lo que está yendo mal.

Por favor, tenga en cuenta que: este paquete depende del X11 de Apple (la versión Mac del sistema de ventanas X). Este está disponible en los DVDs de instalación de OS X.

Cosas opcionales que es posible que Vd. desee instalar:

X-12-ARIMA (ajuste estacional, modelos ARIMA; para Intel Macs)

x12arima-intel.dmg

X-12-ARIMA (ajuste estacional, modelos ARIMA; para PPC Macs)

x12arima-ppc.dmg

TRAMO/SEATS (ajute estacional, detección de outliers, modelos ARIMA)

tramo-seats.dmg

Método alternativo: compilar usando Fink

Si la imagen precompilada no le funciona, o si Vd. quiere más control sobre el proceso de configuración y compilación, Vd puede compilar gretl desde los archivos fuente. Ona manera de hacer esto is por medio del sistema Fink, disponible en http://fink.sourceforge.net/download/.

Además de la instalación básica del propio Fink, para compilar y ejecutar gretl Vd necesitará asegurarse de que los siguientes paquetes de Fink están disponibles (algunos de ellos es posible que ya estén instalados y actualizados, en ese caso Fink le informará de ello). Después de cada paquete, se presenta la instrucción (a ser ejecutada en el símbolo del sistema en un xterm o la aplicación de terminal de Mac) que hará el trabajo:

system-xfree86

 fink install system-xfree86

gtk+2

 fink install gtk+2

gtk+2-dev

 fink install gtk+2-dev

libxml2

 fink install libxml2

gnuplot

 fink install gnuplot

fftw3

 fink install fftw3

En particular, Vd debería asegurarse de que gnuplot esté actualizado (la versión debería ser al menos la 4.0 para funcionar bien con gretl).

En algún tiempo podría Vd necesitar también los siguiente, pero esto puede ser redundante ahora:

dlcompat

 fink install dlcompat

dlcompat-dev

 fink install dlcompat-dev

Variables de entorno

Para compilar gretl usando Fink, Vd necesita asegurarse de que el entorno está correctamente configurado -- antes de ejecutar el guión configure de gretl. Esto implica especificar varias variables de entorno; cómo hacer esto exactamente depende de si su intérprete de instrucciones es tcsh o bash (tcsh era la opción por defecto en OS X antes de Panther, pero creo que bash la opción por defecto bajo Panther). Vd puede ejecutar las siguientes instrucciones una por una o (mejor) ponerlas en el fichero de inicio de su intérprete de instrucciones, .tcshrc o .bashrc (creo) en su directorio personal.

versión tcsh:

setenv CFLAGS -I/sw/include
setenv LDFLAGS -L/sw/lib
setenv CXXFLAGS $CFLAGS
setenv CPPFLAGS $CXXFLAGS
setenv ACLOCAL_FLAGS "-I/sw/share/aclocal"
setenv PKG_CONFIG_PATH "/sw/lib/pkgconfig"

versión bash:

export CFLAGS=-I/sw/include
export LDFLAGS=-L/sw/lib
export CXXFLAGS=$CFLAGS
export CPPFLAGS=$CXXFLAGS
export ACLOCAL_FLAGS="-I/sw/share/aclocal"
export PKG_CONFIG_PATH="/sw/lib/pkgconfig"

Vd también debería asegurarse de ejecuar 'source' para el init.sh suministrado por fink:

source /sw/bin/init.sh

Es posible que Vd desee echar un vistazo a las FAQ de fink.

Configurar y compilar

Una vez que Vd ya tiene las versiones correctas de OS X, X11 de Apple, Fink, y los paquetes requeridos por Fink, y que Vd ha configurado correctamente el entorno, instalar gretl debería ser bastante directo. Espero dejar disponible un paquete de gretl para Fink dentro de poco, pero por ahora Vd tiene que instalarlo desde la fuente. Aquí le explico cómo se hace:

  1. Descargue el paquete fuente:
    curl -O http://ricardo.ecn.wfu.edu/pub/gretl/gretl-VERSION.tar.bz2

  2. Descomprima las fuentes:
    tar xvfj gretl-VERSION.tar.bz2

  3. Cambie al directorio de fuentes de gretl:
    cd gretl-VERSION

  4. Configure el paquete de código fuente:
    ./configure --prefix=/sw --disable-png-comments

  5. Escriba make para compilar gretl. Esto lleva un rato.

  6. Escriba sudo make install para instalar gretl.

Por si puede serle útil, aquí está todo el lote en forma de un guión para el intérprete de instrucciones:

#!/bin/sh
# initialize fink
. /sw/bin/init.sh
# set up the compilation environment
export CFLAGS="-O2 -I/sw/include"
export LDFLAGS=-L/sw/lib
export CXXFLAGS=$CFLAGS
export CPPFLAGS=$CXXFLAGS
export ACLOCAL_FLAGS="-I/sw/share/aclocal"
export PKG_CONFIG_PATH="/sw/lib/pkgconfig"
# download the gretl source
curl -O http://ricardo.ecn.wfu.edu/pub/gretl/gretl-VERSION.tar.bz2
# unpack the source archive
tar xvfj gretl-VERSION.tar.bz2
# move into the gretl source directory
cd gretl-VERSION
# configure the build
./configure --prefix=/sw
# make the program
make
# install gretl (root password needed)
sudo make install

Ahora Vd debería de poder ejecutar gretl escribiendo "gretl" en el símbolo del sistema en un xterm (que se lanza iniciando el X11 de Apple).
