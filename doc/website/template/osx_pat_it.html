Gretl: Gnu Regression, Econometrics and Time-series Library

per Mac OS X

Passo 0

Nel corso della procedura seguente, alcuni passi dipendono dal tipo di Mac che è in uso. Se non si è sicuri a riguardo, basta fare clic sull'icona Apple nell'angolo superiore sinistro dello schermo e selezionare "About this Mac". Apparirà una finestra che contiene il numero di versione di OS X e il tipo di processore, Intel o PowerPC (PPC).

Passo 1

Per usare gretl occorre installare il sistema X11 window. Nelle versioni recenti di OS X questo è già installato, ma se si sta usando una versione precedente può essere necessario dover installare X11.

Per controllare se X11 è installato, basta aprire Finder e aprire Applications, Utilities: se X11 è installato apparirà in fondo alla lista.

Passo 2

Gretl ha bisogno anche di alcune librerie di supporto, ossia GTK e librerie correlate. Se si usa OS X 10.6 (Snow Leopard) o successivo su un Mac Intel, installare GTK usando GTK+.framework.dmg. Se si usa un Mac PowerPC, o una versione precedente di OS X, installare questa versione di GTK (disponibile anche qui).

Passo 3

E ora gretl: scaricare e installare l'ultima versione rilasciata, oppure lo "snapshot" attuale. Quest'ultimo è più aggiornato: spesso contiene correzioni a bug, ma talvolta contiene nuovi bug. Per vedere le novità contenute nello snapshot, si può vedere il Change log di gretl (la voce "in progress").

Finito il download, fare doppio clic sul file dmg.gz per scompattarlo: si otterrà il file dmg; fare doppio clic su di esso per montare l'immagine disco. Aprire l'immagine con Finder e trascinare Gretl su Applications. Attenzione: se si sta aggiornando un'installazione esistente di gretl occorre per prima cosa aprire Applications in Finder e cancellare la vecchia voce Gretl. Fatto questo, è possibile rimuovere l'immagine disco e cancellare i file dmg e dmg.gz.

L'immagine disco di gretl è documentata brevemente nel file README.pdf, che contiene suggerimenti da seguire nel caso si presentino problemi. Se gretl non funziona correttamente su OS X, è possibile fare il debug in questo modo: aprire una finestra di terminale X11, andare nella directory in cui è stato installato il pacchetto (es. cd Gretl.app), andare nella directory Contents/Resources/bin ed eseguire il comando

./gretl

Dovrebbero apparire alcune indicazioni utili per capire cosa c'?che non va.
