Gretl: Gnu Regression, Econometrics and Time-series Library

para Mac OS X com processador PPC

Passo 1: certifique-se que tem X11

Para correr gretl, você tem que ter um sistema de janelas X11 instalado no seu Mac. Isto deverá se automático em OS X recentes, mas se você usa uma versão mais antiga talvez tenha que usar o DVD de instalação do sistema para instalar o X11..

Como saber se o X11 está instalado? Abrir o Localizador e ir a Aplicativos. Dentro de Aplicativos, localize a pasta Utilidades, abrir-la e verificar no fim: se o X11 estiver instalado ele deve aparecer aí como um ícone.

Passo 2: instalar GTK

Algumas bibliotecas adicionais são necessárias para o gretl, nomeadamente GTK e outros relacionados. Instalar GTK usando gtk2-framework.dmg (ou sítio alternativo ).

Passo 3: instalar gretl

Agora para o próprio gretl: obter e instalar a versão mais recente ou o "snapshot" (versão de desenvolvimento). O "snapshot" está mais atualizado: normalmente conterá correções de erros, mas por vezes poderá ter novos erros introduzidos. Para saber o que existe novo no "snapshot", dê uma vista de olhos no Registo/Registro de alterações do gretl (na entrada "in progress").

versão mais recente (SHORTDATE)

gretl-VERSION-ppc.dmg

OU "snapshot" atual

gretl-ppc.dmg

Depois de obter o gretl, faça um duplo-clique no arquivodmg para montar a imagem de disco. Abrir a imagem de disco no Localizador e arrastar Gretl para dentro de Aplicativos. Abrir a imagem de disco no Finder e arrastar Gretl para dentro de Applications. Mas note: se você estiver atualizando uma instalação do gretl, você deverá primeiro abrir Aplicativos com o Localizador e apagar o item do Gretl antigo. Depois de ter instalado o gretl você pode ejetar a imagem de disco e apagar o arquivo dmg.

Problemas?

A imagem de disco do gretl disk image está documentada em README.pdf, que contém algumas sugestões para despiste de problemas. Resumidamente, se você quiser despistar problemas num gretl não funcional em OS X, tente isto: abrir uma janela de terminal X11; mudar de diretório (cd) para Gretl.app (o local onde instalou o pacote); mudar para o diretórioContents/Resources/bin e rodar o comando

./gretl

Isto deve mostrar alguma informação sobre o que está acontecendo de errado.

Extras opcionais

Você pode querer instalar X-12-ARIMA e/ou TRAMO/SEATS. Estes são programas especializados para análise de séries temporais com sazonalidade, em versões compatíveis com gretl.
