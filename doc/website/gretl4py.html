About

gretl4py is a python package designed to meet the needs of econometricians and applied economists who wish to use python in their research. It can be broadly viewed as a complement to the statsmodels package, with a much more specific orientation to the inference methods commonly employed in applied economics.

gretl4py provides an interface to libgretl for the Python language (3.11, 3.12, and 3.13 were tested). Libgretl is the underlying engine for the gretl program. At present, the focus is mostly on the APIs implementing estimating methods, but a much larger array of facilities is planned.

gretl4py consists of the following elements:
  • binary bindings written in C++,
  • a Model class implemented in Python,
  • a set of utility functions implemented in Python.
The source code is available as git repository here.

Preliminaries

Preparation

Before installing gretl4py, please make sure that the following components are installed:
  • One of the supported versions of Python.
  • The following Python packages: numpy, and tabulate (not required).
  • macOS or Linux users: gretl version 2024c or higher:
    • Linux: gretl package (Debian/Ubuntu/Fedora).
    • macOS: see this page.
  • MS Windows users: gretl version 2025a or higher (snapshots may be downloaded from here).

Download

You may download gretl4py as zip archive from here, as snapshot directly from the SourceForge website.

  • MS Windows: gretl4py_for_windows.zip
  • macOS: in progress
  • Linux: gretl4py_for_linux.zip

Installation

MS Windows and MacOS

The easiest way to install gretl4py on a MS Windows or macOS machine is by executing the install.py script located in the gretl4py top-level directory using the desired Python interpreter:

  • On MS Windows: python.exe install.py
  • On macOS: python3 install.py

The script assumes that python.exe (on MS Windows) or python3 (on macOS) is available in the system's PATH.

The install.py script copies the following files to the user site-packages directory, as determined by the command python.exe -m site --user-site on MS Windows or python3 -m site --user-site on macOS:
  • __init__.py, gretl4py_addons.py, and gretl4py_classes.py from the [gretl] directory.
  • _gretl.cp3xx-win_amd64.pyd (for MS Windows) or _gretl.cpython-3xx-darwin.so (for macOS) from the [builds/3.xx] directory.
Here, 3.xx refers to the version of Python used to run the install.py script.

Linux

Linux users have two options:

  • Build gretlp4y yourself (recommended)

    Instructions are here (section 2.3). Apart from a C++ compiler (with support for c++17 standard), the following development packages are needed (2024c or higher):

    • Debian/Ubuntu: libgretl1-dev, python3-dev
    • Fedora: gretl-devel

  • Install gretlp4y precompiled binaries (experimental)

    Execute the install.py script located in the gretl4py top-level directory using the desired Python interpreter.

Running gretl4py

To verify if gretl4py was installed correctly please run the following code snippet:

python3 -c 'import gretl; gretl.about(1)'

This should give you some text output on
  1. your gretl4py build;
  2. your system, as provided by the gretl $sysinfo accessor;
  3. a few libgretl built-in strings;
  4. the simdjson architecture.

Note: On MS Windows, when gretl4py is loaded, you may see some warnings about the missing 'dark2' Gnuplot style. Don't worry, this is not a problem.
