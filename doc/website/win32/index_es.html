versión para win32

Descarga

gretl para win32 viene en un fichero ejecutable auto-extraíble, gretl_install.exe (feb 2, 2004, aproximadamente 7MB). Vd. sólo tiene que descargarlo y ejecutarlo. Durante el proceso de instalación se le preguntará por el directorio donde desea instalar el programa (por defecto es c:\userdata\gretl). Es posible que desee también instalar los programas de corrección estacional X-12-ARIMA o TRAMO/SEATS en una forma adecuada para usarlo junto con gretl: para esto descargue los programas de instalación x12a_install.exe o ts_install.exe.

El manual de gretl (en Inglés) se encuentra aparte en un fichero de formato PDF: manual.pdf. Éste está formateado para el tipo de papel 'US letter'; también hay una versión A4: manual-a4.pdf. Si Vd. no tiene el programa para leer ficheros PDF, puede descargarlo de forma gratuíta desde el sitio web de Adobe.

Los ficheros ejecutables se compilaron bajo GNU/Linux usando mingw32 y el gtk para win32 de Tor Lillqvist. El programa de instalación, también gratuíto, es cortesía de Jordan Russell.

Actualizando gretl

Cuando se actualiza gretl, teniendo instalada una versión previa, probablemente no sea mala idea ejecutar primero el programa desinstalador unins000.exe, que debería estar en el directorio principal de gretl. Esto evitará que gretl funcione mal debido al uso de DLLs antiguos o inscripciones anteriores en el registro de Windows. Nótese que al ejecutar el desinstalador no se tocará ningún fichero de usuario que Vd. haya creado en el directorio gretl/user, pero se borrará la configuración personal que haya determinado utilizando el menú 'preferencias', así que tendrá Vd. que restablecerla cuando ejecute la nueva versión del programa.

Construyendo gretl para win32

Si Vd. desea construir gretl para Windows Vd. mismo, descargue el paquete de fuentes de gretl (vaya aquí para encontrar el más reciente) y eche un vistazo al subdirectorio win32. Aquí encontrará el archivo 'Makefile' para la versión Windows: actualmente está diseñado para cross-compilar gretl en Linux pero no debería ser muy difícil modificarlo para compilar en Windows, especialmente si Vd. utiliza el compilador mingw32.

gnuplot

gretl llama a gnuplot para generar los gráficos. Hay una copia de gnuplot en el propio fichero de instalación de gretl.
