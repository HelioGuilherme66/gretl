Colecções de Dados e Scripts

Se utiliza o gretl num contexto de ensino, você pode estar interessado em preparar uma colecção de dados e scripts que estejam relacionados com o seu curso, numa forma em que os alunos os possam observar e aceder a eles com facilidade. Isto é possível fazer a partir da versão 1.2.1 e superiores.

As matérias podem ser arranjadas de modo que um ficheiro de dados personalizado apareça nas opções quando um utilizador selecciona "Ficheiro, Abrir Dados, ficheiro de amostras", ou clica no ícone "Pasta" na barra de ferramentas do gretl.

De modo semelhante, você pode preparar uma colecção de scripts de modo a que apareça quando o utilizador selecciona no menu, "Ficheiro, Abrir ficheiro de comandos, ficheiro de prática".

Quando se acede a um dos items de menu mencionado acima, o gretl procura por ficheiros de colecções válidos nestas localizações: a directoria de dados do gretl "system", a directoria de sistema de scripts do gretl, a directoria do utilizador de gretl, e todas as subdirectorias do primeiro nível destas. Para referência, os valores usuais para estas directorias são:

localização Linux por omissão Windows por omissão

system data dir

 /usr/share/gretl/data c:\userdata\gretl\data

system script dir

 /usr/share/gretl/scripts c:\userdata\gretl\scripts

user d/home/me/gretl

 c:\userdata\gretl\user

Todas as colecções válidas serão adicionadas à janela de selecção. Então o que constitui um ficheiro de uma colecção válida? Comporta tanto um conjunto de dados no formato XML do gretl (com a extensão .gdt) como um conjunto de ficheiros script contendo comandos do gretl (com a extensão .inp), e em ambos os casos, acompanhados por um "ficheiro mestre" ou catálogo. A distribuição corrente do gretl contém vários exemplos de catálogos, por exemplo o ficheiro descriptions na subdirectoria "misc" na directoria dos dados do gretl e ps_descriptions na subdirectoria "misc" da directoria de scripts do gretl.

Recomendo que se estiver a adicionar a sua própria colecção, que os catálogos de dados se chamem "descriptions" e os catálogos de script com o nome "ps_descriptions". Em ambos os casos este ficheiros devem ser colocados (juntamente com os de dados ou scripts) numa subdirectoria própria para eles (por exemplo /usr/share/gretl/data/osmeusdados ou c:\userdata\gretl\data\osmeusdados).

A sintaxe dos ficheiros de descrição (em texto simples) é muito simples. Eis, por exemplo, algumas das primeiras linhas do ficheiro de catálogo de dados do gretl, "misc":

 # Gretl: vários exemplos ilustrativos de ficheiros de dados
 "arma","dados artificiais para o exemplo de script ARMA (autoregressão com média móvel)"
 "ects_nls","Exemplo de Mínimos Quadrados Não-Lineares"
 "hamilton","Preços e taxa de câmbio, E.U.A. e Itália"

A primeira linha, que tem que iniciar com o símbolo cardinal, contém um pequeno nome, neste caso "Gretl", que aparecerá como o nome desta colecção no separador da janela de selecção de dados, seguido de um símbolo dois pontos, e seguido por uma breve e opcional descrição da colecção.

As linhas subse vírgula e entre aspas. O primeiro é o nome do ficheiro de dados (sem a extensão .gdt) e o segundo uma curta descrição do conteúdo desse ficheiro de dados. Deve existir uma dessas linhas para cada ficheiro de dados na colecção.

Um ficheiro de catálogo para uma colecção de scripts, parece muito semelhante, excepto que existem três campos nas linhas dos ficheiros: um nome de ficheiro (sem a extensão .inp), uma breve descrição do assunto de econometria ilustrado no exemplo, e uma pequena indicação da natureza dos dados utilizados. Mais uma vez, eis algumas das primeiras linhas do catálogo de scripts de "misc" do gretl:

 # Gretl: vários scripts de exemplos
 "arma.inp","modelação ARMA","dados artificiais"
 "ects_nls","Mínimos Quadrados Não-Lineares (Davidson)","dados artificiais"
 "leverage","Observações Influenciadoras", "dados artificiais"
 "longley","Multicolinearidade","emprego E.U.A."

Estes são os passos para colocar a sua própria colecção de dados disponível no gretl:

  1. Reunir os dados, em qualquer formato que seja conveniente.

  2. Converter os dados para o formato do gretl e guardar como ficheiros .gdt. Provavelmente é mais fácil converter os dados importando-os para o gretl a partir de texto simples, CSV (Valores Separados por Dois Pontos) ou em formato de folha de cálculo (MS Excel ou Gnumeric) e depois gravar como gdt. Poderá pretender acrescentar descrições das variáveis individuais (a função no menu, "Variável, Editar atributos"), e acrescentar informação sobre a origem dos dados (a função "Dados, Editar informação").

  3. Escrever um ficheiro de descrições ("descriptions") para a colecção usando um editor de texto.

  4. Colocar os ficheiros de dados e o de descrições numa subdirectoria da directoria de dados do gretl (ou directoria do utilizador).

Seem proprietários, gostava de sugerir que fornecesse a colecção para ficar disponível como um pacote opcional do gretl ("optional extra").
