\section{Características principales}
\label{features}

\app{Gretl} es un paquete de cálculo econométrico, e incluye una
biblioteca compartida, un programa cliente de línea de
instrucciones y
un interfaz gráfico para el usuario (GUI).

\begin{description}

\item[Fácil de usar] \app{Gretl} ofrece un interfaz intuitivo para el
  usuario; es muy fácil de instalar y de usar para realizar análisis
  econométrico. El paquete está muy ligado a los manuales de
Econometría de Ramu Ramanathan, Jeffrey Woolridge y James Stock y
Mark Watson lo
cual permite ofrecer muchos archivos de datos para ejercicios
prácticos así como ficheros con lotes de instrucciones (scripts).
Estos son fácilmente accesibles y vienen acompañados de muchos
comentarios.

\item[Flexibilidad] \app{Gretl} ofrece la posibilidad de elegir entre
  un enorme rango de opciones; desde apuntar y hacer click en modo
  interactivo hasta la utilización de procesos por lotes, resultando
  fácil además hacer diferentes combinaciones entre las mismas.

\item[Multiplataforma] La plataforma nativa de \app{Gretl} es Linux,
  pero también está disponible para MS Windows y Mac OS X y debería
funcionar en cualquier sistema tipo Unix que contenga
  los archivos de biblioteca básicos adecuados (véase el
  apéndice~\ref{app-technote}).

\item[Código abierto] El código fuente de \app{Gretl} se encuentra a
  la total disposición de cualquiera que esté interesado en revisar,
  completar o extender el programa. 

\item[Sofisticación] \app{Gretl} ofrece una completa gama de
  estimadores de mínimos cuadrados, incluyendo \emph{mínimos cuadrados
    en dos etapas} y mínimos cuadrados no lineales. También ofrece
    varios estimadores de máxima verosimilitud específicos (por
ejemplo, logit, probit, tobit) y desde la versión 1.5.0, un estimador
de máxima verosimilitud general. El programa es capaz de estimar
sistemas de ecuaciones simultáneas, GARCH, ARMA, VAR y modelos de
corrección de error vectoriales.

\item[Precisión] \app{Gretl} ha sido contrastado con
los conjuntos de datos de referencia de NIST, obteniendo muy buenos
resultados. Véase el Apéndice~\ref{app-accuracy}.

\item[Uso de Internet] \app{Gretl} tiene acceso a un servidor en Wake
  Forest University desde el cual puede descargar algunas bases de
  datos. La versión para MS Windows incorpora un programa de
  actualización que detecta la disponibilidad de nuevas versiones de
  \app{Gretl} y ofrece la opción de actualización de manera
  automática.

\item[Internacional] \app{Gretl} puede producir sus resultados en
  Inglés, Español, Francés,  Italiano, Polaco o Alemán dependiendo
del idioma nativo del ordenador.

\end{description}

\section{Instalación de los programas}
\label{install}

\subsection{Linux}
\label{linux-install}

En la plataforma Linux\footnote{En este
manual usaremos ``Linux'' para  referirnos al sistema operativo
GNU/Linux.  Lo que aquí se dice  acerca de Linux es, en su mayor
parte, válido también para otros  sistemas similares a Unix, aunque
puede resultar necesaria alguna que otra modificación local.} tenemos
la opción de compilar el
código \app{Gretl} o usar un paquete ya precompilado. Hay
paquetes precompilados disponibles en el formato \app{rpm}
(aconsejable para Red Hat Linux y sistemas relacionados con este) y
también en el formato \app{deb} (Debian GNU/Linux). En el caso de que
el usuario prefiriera construir uno por sí mismo, (o si el sistema
Unix no dispone de paquetes ya construidos), deberá seguir los
siguientes pasos.


\begin{enumerate}
\item Descargar la última distribución de \app{Gretl} desde
  \href{http://Gretl.sourceforge.net/}{Gretl.sourceforge.net}.
\item Descomprimir (unzip+untar) el paquete. En un sistema que
  disponga de utilidades GNU, la instrucción sería \cmd{tar xvfz
    Gretl-N.tar.gz} (reemplazar \cmd{N} con el número de versión
  específico del archivo que hemos bajado en el paso 1).
\item Ir al directorio de \app{Gretl} creado en el paso 2 (e.g.
  \verb+Gretl-1.1.5+).
\item Entonces, la rutina básica de compilación es
        
\begin{code}
    ./configure 
    make 
    make check 
    make install
\end{code}

Sin embargo, antes de empezar, es conveniente leer el archivo
\verb+INSTALL,+ y/o hacer 
\begin{code}
    ./configure --help
\end{code}
para ver las opciones disponibles. Una de las opciones que el usuario
puede querer ajustar es \cmd{--prefix}.  La instalación se hace en
\verb+/usr/local+ por defecto, pero es posible cambiarlo. Por ejemplo,
\begin{code}
    ./configure --prefix=/usr
\end{code}
pondrá todo bajo el directorio \verb+/usr+. En el caso de que algún
archivo de biblioteca necesario no se encuentre en el sistema, y por
lo tanto falle todo el proceso de instalación, consúltese el
Apéndice~\ref{app-technote}.

\end{enumerate}

\app{Gretl} ofrece soporte para el escritorio de
\app{gnome}. Para utilizarlo, se compila el programa en la forma
descrita anteriormente.  Si se desean suprimir los rasgos específicos
de gnome, es preciso pasar la opción \verb+--without-gnome+
directamente a \cmd{configure}.

\subsection{MS Windows}
\label{windows-install}

La versión para MS Windows se distribuye en un archivo
autoejecutable. La instalación consiste simplemente en bajar
\verb+Gretl_install.exe+ y ejecutar este programa. Durante el
proceso de instalación se le preguntará por un lugar donde instalar el
paquete (por defecto se instala en \verb+c:\userdata\Gretl+).

\subsection{Actualizaciones}
\label{updating}

En el caso de que el ordenador este conectado a Internet, \app{Gretl}
cuenta con la posibilidad de conectarse a su sitio web, en la
Universidad de Wake Forest, al iniciar el programa para ver si hay
una nueva versión disponible. Si la hay, se abre una ventana
informando al usuario de este hecho. Si queremos activar esta opción,
hay que marcar la opción ``Informar sobre actualizaciones de
\app{Gretl}'' en el menú ``Archivo, Preferencias, General '' de
\app{Gretl}.

La versión para MS Windows adelanta un paso más: da la opción de
actualización automática, si se desea. Para seleccionar esta opción,
síganse las instrucciones en la ventana emergente: cerrar
\app{Gretl} y a continuación ejecutar el programa llamado ``\app{Gretl
  updater}'' (que se encuentra, junto con el programa principal de
\app{Gretl}, bajo Programas en el menú de inicio de Windows). Una vez
terminado el proceso de actualización, se puede reiniciar \app{Gretl}.
