\section{Introducción}
\label{cmd-intro}

Las instrucciones definidas más abajo pueden ejecutarse de forma interactiva en
el programa cliente de lineas de instrucciones, o en la ventana de la consola del
programa GUI. Y también se pueden colocar en un archivo de ``guion'' o de
procesamiento por lotes para su ejecución interactiva.
    
Más abajo se utilizan las siguientes convenciones en la notación:
    
\begin{itemize}

\item Una \texttt{fuente typewriter} se utiliza para aquellas cosas que escribirías
  tal cual, y también para nombres internos de variables.

\item Los elementos en una \textsl{fuente cursiva} son marcadores de posición:
  deberías cambiarlos por algún sustituto en concreto. Por ejemplo, podrías poner
  \texttt{ingreso} en lugar del genérico \textsl{varx}.

\item La estructura \texttt{[} \textsl{arg} \texttt{]} significa que el argumento
  \textsl{arg} es opcional: podrías indicarlo o no (pero en cualquier caso no
  pongas los corchetes).

\item La frase ``instrucción de estimación'' significa una instrucción que genera
  estimaciones para un modelo dado, por ejemplo \cmd{ols}, \cmd{ar} o
  \cmd{wls}.

\end{itemize}

En general, cada linea de un guion de instrucciones debiera contener una única
instrucción de \app{GRETL} completa. Sin embargo, hay dos formas de continuar
una larga instrucción que va de una linea de entrada a otra. Primera, si el último
carácter de una linea (que no sea un espacio) es una barra diagonal invertida,
esto se considera un indicador de que la instrucción continúa en la siguiente
linea. Además, si la coma es un carácter válido en una instrucción dada
(por ejemplo, como separadora entre argumentos de una función, o como
signo de puntuación e la instrucción \texttt{printf}), entonces una coma al final
también indica continuación. Para enfatizar este punto: una barra diagonal
invertida se puede insertar ``arbitrariamente'' para indicar una continuación,
pero una coma funciona con esa capacidad solo si es sintácticamente válida
como parte de la instrucción.
